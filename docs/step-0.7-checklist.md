## ✅ 相关功能测试（10分钟）

### 1. escapeHtml 测试
**测试位置**: 书签列表、文件夹列表

- [ ] 查看书签列表，书签标题中的特殊字符（`<` `>` `&` `" `'`）正确转义
- [ ] 查看文件夹名称，特殊字符正确转义
- [ ] **安全性**: 无 XSS 风险（标题中包含 `<script>` 不会执行）

**测试方法**:
1. 找一个标题包含特殊字符的书签或文件夹
2. 检查元素，确认 `&` 显示为 `&amp;`，`<` 显示为 `&lt;` 等

### 2. truncateUrl 测试
**测试位置**: 书签列表的 URL 显示

- [ ] 书签列表中，长 URL 被正确截断
- [ ] 截断后显示 `...` 后缀
- [ ] 鼠标悬停可以看到完整 URL（title 属性）

**测试方法**:
1. 找一个 URL 较长的书签
2. 确认 URL 显示为 `https://example.com/very/long...` 格式

---

### 3. isValidUrl 测试
**测试位置**: 添加/编辑书签对话框

- [ ] 打开"添加书签"对话框
- [ ] 输入有效 URL（`https://example.com`），可以保存
- [ ] 输入无效 URL（`not-a-url`），**提示格式错误或阻止保存**
- [ ] 输入 `http://` 开头的 URL，可以保存
- [ ] 输入其他协议（`ftp://`），**应该阻止保存**

**测试方法**:
1. 点击右上角收藏按钮
2. 测试不同 URL 格式

---

### 4. normalizeUrl 测试
**测试位置**: 去重功能

- [ ] 点击任务面板的"🗑️ 去重"按钮
- [ ] 去重功能正常启动
- [ ] 能正确识别带跟踪参数的重复 URL
  - `https://example.com?utm_source=google` 和 `https://example.com` 被识别为重复
- [ ] 能正确识别带末尾 `/` 的重复 URL
  - `https://example.com/` 和 `https://example.com` 被识别为重复

**测试方法**:
1. 准备两个书签：
   - 书签A: `https://example.com`
   - 书签B: `https://example.com/?utm_source=google`
2. 点击去重按钮
3. 确认这两个书签被识别为重复

---

### 5. isInBookmarksBar 测试
**测试位置**: 去重功能（保留书签栏中的书签）

- [ ] 去重时，优先保留书签栏中的书签
- [ ] 删除书签时，书签栏中的书签有特殊标识
- [ ] 去重对话框正确显示书签路径

**测试方法**:
1. 准备两个重复书签：
   - 书签A（在书签栏中）
   - 书签B（在其他文件夹中）
2. 执行去重
3. 确认保留的是书签A（书签栏中的）

---

### 6. highlightKeywords 测试
**测试位置**: 搜索结果

- [ ] 在搜索框输入关键词
- [ ] 搜索结果中，关键词被正确高亮
- [ ] 高亮显示为黄色背景（`<mark>` 标签）
- [ ] **支持多关键词**：输入 `javascript react`，两个词都被高亮
- [ ] **搜索操作符不参与高亮**：输入 `tag:技术 site:example.com`，这些前缀不被高亮

**测试方法**:
1. 搜索 `javascript`
2. 确认结果中的 `javascript` 被高亮显示
3. 搜索 `javascript react`
4. 确认两个关键词都被高亮
5. 搜索 `tag:技术`
6. 确认 `tag:` 不被高亮，只有 `技术` 被高亮
