## 发布流程

### 阶段 1: 预检查（发布前 1 周）

1. 完成上述所有检查清单项目
2. 修复所有 P0 和 P1 级别 Bug
3. 更新所有文档
4. 准备商店素材

### 阶段 2: 测试验证（发布前 3 天）

1. 在 Chrome 和 Edge 上完整测试
2. 邀请 beta 用户测试
3. 性能测试和压力测试
4. 安全审查

### 阶段 3: 发布准备（发布前 1 天）

1. 创建发布分支
2. 更新版本号
3. 生成发布包
4. 准备发布说明

### 阶段 4: 正式发布（发布日）

1. 提交到 Chrome Web Store
2. 提交到 Edge Add-ons
3. 发布 Git Release
4. 发布公告

### 阶段 5: 发布后（发布后 1 周）

1. 监控用户反馈
2. 修复紧急 Bug
3. 收集改进建议
4. 规划下一版本

## 风险评估

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| Chrome Web Store 审核不通过 | 高 | 低 | 提前审查权限和隐私政策 |
| 重大 Bug 未发现 | 高 | 中 | 充分的 Beta 测试 |
| 性能问题 | 中 | 低 | 性能测试和优化 |
| 用户数据迁移失败 | 高 | 低 | 数据库迁移测试 |
| API 兼容性问题 | 中 | 低 | 多 API 提供商测试 |
