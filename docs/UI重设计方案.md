# Smart Bookmarks — 整屏 UI 重设计方案

> 版本：v2.0  
> 日期：2026-03-12  
> 状态：待实现

---

## 一、设计目标

将原 580×660 px 弹窗界面重构为**浏览器全屏标签页（Full-Tab Page）**，提供类似原生书签管理器的沉浸式使用体验，同时保留全部现有功能。

---

## 三、各区域详细设计

### 3.1 顶部 Header

```
[🔖 Smart Bookmarks]  [🔍 搜索框（居中，占主要宽度）]  [⚙️][📥][📤][🔄]
```

- 左侧：Logo + 产品名
- 中部：搜索框（全局语义搜索，支持 `tag:标签` / `site:域名`），宽度自适应
- 右侧工具栏按钮（图标形式，hover 显示 tooltip）：
  - ⚙️ 设置
  - 📥 导入
  - 📤 导出
  - 🔄 同步

高度：52 px，渐变背景（indigo → violet）

---

### 3.2 左侧目录树（Sidebar）

**功能**：
- 展示所有文件夹/分类的树形结构
- 点击文件夹时，右侧主区域切换到该文件夹内容
- 当前激活的文件夹高亮显示
- 折叠/展开子目录
- 顶部固定项：「全部收藏」「⭐ 最近」「⚠️ 失效链接」「🏷️ 标签视图」

**交互**：
- 分割线可拖拽，改变左右宽度比例（最小左 180px，最小右 400px）
- 拖拽宽度持久化到 `chrome.storage.local`
- 文件夹节点右键菜单：新建子文件夹、重命名、删除
- 底部状态栏：显示总条目数

**样式**：
```
背景色：bg-slate-50（略深于主区域 bg-white）
宽度：260px（用户可拖拽调整）
字体：text-sm
激活项：bg-indigo-50 text-indigo-700 font-medium left-border 3px indigo-500
悬停项：bg-slate-100
缩进：每层 16px（通过 padding-left）
```

---

### 3.3 右侧主区域

#### 3.3.1 面包屑导航

```
📁 收藏夹  >  人工智能  >  ...
```

- 点击任意层级可跳转
- 最右侧显示当前文件夹下的条目总数

#### 3.3.2 内容列表

每行分两种类型：

**文件夹行**：
```
[📁 图标]  文件夹名称                    [子项目数量 badge]  [⋮]
```

**书签行**：
```
[网站 favicon / 默认图标]  标题文字（bold）               [⋮]
                          概述描述文字（两行截断，灰色）
                          [标签1] [标签2]  ·  domain.com · ⚠️失效
```

- 列表项 hover：显示淡蓝色背景 + 右侧 `⋮` 菜单按钮
- 标题点击：新标签页打开链接
- 每行高度：书签约 72px，文件夹约 44px
- 失效链接：行背景 `bg-red-50`，标题旁显示 `⚠️` 标记
- 支持键盘方向键上下导航

#### 3.3.3 行内上下文菜单（⋮）

点击 `⋮` 或右键点击书签行时出现：

```
┌─────────────────────────┐
│  修改                    │
│  删除（红色）             │
│  ─────────────────────  │
│  剪切                    │
│  复制                    │
│  粘贴                    │
│  ─────────────────────  │
│  重新生成概述             │
│  ─────────────────────  │
│  在无痕窗口中打开          │
│  在新标签页分组打开        │
│  在新标签页中打开          │
│  在新窗口中打开            │
└─────────────────────────┘
```

- 自动检测边界，防止超出视口
- 点击外部自动关闭
- 支持键盘 Escape 关闭

│  名称                                  │
│  [输入框：书签标题              ]      │
│                                        │
│  网址                                  │
│  [输入框：https://…            ]      │
│                                        │
│  概述                   [🔄 重新生成]  │
│  [多行文本框：                       ] │
│  [                                   ] │

### 3.5 右下角功能面板（Task Panel）

浮动面板，默认折叠，有任务运行时自动展开：

```
┌────────────────────────────────────────┐ ← 可折叠
│  任务面板                    [─] [×]   │
│  ──────────────────────────────────── │
│  🤖 AI 分析        [▶ 开始] [🔬 调试]  │
│  进度：████████░░  80%  320/400       │
│  批次信息：正在处理第 8 批…            │
│  ──────────────────────────────────── │
│  ⚠️ 链接检测       [▶ 检测] [✕ 取消]  │
│  进度：████░░░░░░  40%  162/400       │
│  ETA：约 2 分钟                       │
└────────────────────────────────────────┘
```

- 位置：`fixed bottom-4 right-4`
- 宽度：360px
- 无任务时：只显示两个启动按钮（小图标模式）
- 有任务运行时：自动展开，显示进度条
- 关闭按钮（×）：折叠面板但不取消任务
- 取消按钮：取消当前任务

## 四、状态管理

```js
const state = {
  // 导航状态
  currentFolderId: 'root',    // 当前浏览的文件夹 ID
  breadcrumb: [],              // 面包屑路径
  sidebarWidth: 260,           // 左侧栏宽度（持久化）
  taskPanelExpanded: false,    // 任务面板是否展开

  // 数据
  bookmarks: [],
  categories: [],
  tags: [],

  // 搜索
  searchTerm: '',
  searchResults: null,         // null = 不在搜索模式

  // 任务状态
  isAnalyzing: false,
  isChecking: false,
  analyzeProgress: { current: 0, total: 0, message: '' },
  checkProgress: { completed: 0, total: 0, brokenCount: 0, eta: '' },

  // UI 状态
  selectedItem: null,
  contextMenuTarget: null,
  editingItem: null,           // 正在编辑的书签
}
```

## 七、响应式 & 性能

- **虚拟滚动**：当收藏数量 > 500 时，仅渲染可视区域内的条目（IntersectionObserver 实现）
- **防抖搜索**：搜索输入 150ms debounce
- **懒加载 favicon**：`<img loading="lazy">` + onerror fallback
- **侧栏宽度持久化**：`chrome.storage.local.set({ sidebarWidth })`
